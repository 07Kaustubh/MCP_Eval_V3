## Sync workflow

1. Diff upstream `MCP_Eval_V3_BrookField/Docs/8_QC_Spec_Doc2.md` against this file's most recent row.
2. For each NEW upstream changelog entry, append a row here AND update the corresponding `Evals/*.md` / `Docs/` content.
3. Run `python3 Validators/check_eval_hashes.py --update` after the sync so the pinned baseline reflects the new state.
4. Update the AGENTS.md "Pipeline Deviations from Eval Specs" table if the new rule conflicts with an existing pipeline interpretation.
5. Flip `pending verification` rows to `ingested in v<NN>` once the corresponding gate is confirmed present in `Evals/`.
