# PIPELINE S4 — Verifier-Fails Analysis

Triggered by: `PIPELINE S4 — Tasks/<TASK_DIR>` followed by paste of verifier fails + trajectories already deposited into `Agent_Responses/`.

## What this phase does

For each rubric that failed across runs, classify the failure: rubric-invalid, judge-error, or legitimate model failure. Produce AF justifications for legitimate failures and fix-lists for invalid rubrics or judge errors.

## Procedure

1. **Build the rubric x run matrix.** For each rubric in `7_Rubrics.json` and each run in `Agent_Responses/`, mark pass / fail / not-evaluated. Identify rubrics that failed all completed runs (AF rubrics) and rubrics that failed some runs (partial fails).

2. **For each failing rubric, classify into one of three buckets:**

   ### Bucket 1 — Rubric Invalid
   The rubric is the problem. Signs:
   - Outcome too strict (expects exact value, agent reported equivalent rounded value).
   - Outcome bundles two independent facts that pass / fail separately.
   - Process rubric punishes a valid alternative path.
   - Process rubric covers a property an Outcome already proves.
   - Phrasing fails a valid agent expression (e.g., expects exact phrase, accepts no synonyms).
   Action: write the fix to `_aux/Council_Reports/S4_fixes.md` with a before / after and reasoning.

   ### Bucket 2 — Judge Error
   The rubric is correct. The agent's trajectory shows it actually satisfied the rubric. The judge misread. Action: log to `_aux/Council_Reports/S4_judge_errors.md` with the trajectory citation (run #, tool call #, parameter value).

   ### Bucket 3 — Legitimate Model Failure (AF Justification)
   The rubric is correct. The agent really did fail. This is desired difficulty. Action: write the AF justification using the template in `Reference/Linter_Playbook.md` — two to four sentences, no em-dashes, no references to guides / specs / frameworks, first-person, cites the concrete data fact and the specific reasoning gap. Save to `_aux/Council_Reports/S4_AF_justifications.md`.

3. **Confirm classifications against the universe.** For every Bucket 1 / Bucket 2 classification, re-grep `_aux/Universe_Split/` to confirm the rubric value vs the agent's trajectory value. Council A handles this if any classification is uncertain.

4. **Update the Hardness Plan calibration.** Compare the stump hypothesis predictions in `_aux/Hardness_Plan.md` against the actual AF rubrics:
   - Predicted + actually failed → confirm in `Tasks/_meta/Stump_Hypotheses.md`.
   - Predicted + did not fail → over-predicted (lever was weaker than expected).
   - Not predicted + failed → under-predicted (which lever did we miss?).
   Append the calibration delta to `Tasks/_meta/Hardness_Patterns_Log.md`.

5. **Produce the verdict report `_aux/Council_Reports/S4_verdict.md`:**

   ```markdown
   # Verifier Fails — S4 verdict

   ## Run matrix
   <rubric × run table>

   ## Classifications
   - Bucket 1 (rubric invalid): <count> rubrics → see S4_fixes.md
   - Bucket 2 (judge error): <count> rubrics → see S4_judge_errors.md
   - Bucket 3 (legitimate AF): <count> rubrics → see S4_AF_justifications.md

   ## Hardness calibration
   - Stump hypothesis hit rate: <hits>/<total>
   - Lessons for next task: <one paragraph>

   ## Action items
   - <fix Bucket 1 rubrics in 7_Rubrics.json>
   - <ship Bucket 3 AF justifications back to the platform>
   - <appeal Bucket 2 judge errors if the platform supports it>
   ```

## Exit criteria

- Every failing rubric classified into exactly one bucket.
- Bucket 1 fixes drafted in `_aux/Council_Reports/S4_fixes.md`.
- Bucket 3 AF justifications drafted in `_aux/Council_Reports/S4_AF_justifications.md` following the strict style (concise, human, no em-dashes, no references to guides).
- `_aux/Council_Reports/S4_verdict.md` exists with the matrix + classifications.
- `Tasks/_meta/Stump_Hypotheses.md` updated.
- `Tasks/_meta/Hardness_Patterns_Log.md` updated.

## Next step

User applies Bucket 1 fixes to `7_Rubrics.json`, submits Bucket 3 AF justifications to the platform, and re-runs verifier if fixes were made. If everything is shipped: EXIT.
