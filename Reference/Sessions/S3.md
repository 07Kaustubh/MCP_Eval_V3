## Procedure

1. **Read every V3 reference rubric file in full.** The 4 sample tasks collectively contain 59 outcome rubrics, zero process. Internalize the phrasing patterns, evidence shapes, and the use of "(or similar)" and "approximately".

2. **Outcome first.** Walk the OE list. For every write action create an Outcome 1.1 rubric. Add 1.2 when the write has content requirements beyond what 1.1 covers. Walk the prompt for every fact the user asked to be told directly and create an Outcome 2.1 rubric. Atomic rubrics for multi-item write actions — one per item grounded in ground truth, never "at least N".

3. **Three-condition test for any Process candidate.** A Process rubric is only valid when ALL THREE hold:
   - Required by every valid solution path (phrased broadly enough to allow alternatives).
   - A stricter Outcome rubric cannot capture the same requirement.
   - Describes a verification, not an execution trace.
   Default expectation: **zero process rubrics**. All 4 V3 reference tasks have zero.

4. **Pick the schema and stay consistent.** Either nested (`{id, title, annotations: {evidence, justification, rubric_category}}`) or flat (`{title, category, justification, evidence}`). Whichever shape the surrounding task ecosystem uses — the validator accepts both.

5. **Run the validator (cheap gate).**
   ```
   python Validators/validate.py --phase rubrics --task Tasks/<TASK_DIR>
   ```
   Read `_aux/Validator_Reports/rubrics.md`. Fix every fail. The validator catches: schema, agent-centric phrasing, tool-name leak in title, "at least N" without prompt mandate, outcome > process count, grounded-sweep mismatches.

6. **Spawn Council A — Grounding.** The sub-agent walks every rubric title, extracts every concrete value (email, amount, ID, account, retention code, classification, persona name, period id, date), and greps `_aux/Universe_Split/`. Block on any ungrounded value. Output to `_aux/Council_Reports/S3_A_grounding.md`.

7. **Spawn Council B — Adversarial QC (the heaviest single sub-agent call in the pipeline).** Use `ultrabrain` for this one. Tasks:
   - Score every applicable rubric sub-dim from `Docs/7_QC_Spec_Doc1.json` (Atomicity, Self-Containment, Completeness, Flexibility, Accuracy, Category Balance, Agent-Centric Phrasing).
   - **Adversarial alt-path:** sketch a valid agent trajectory that satisfies the prompt intent but fails an Outcome rubric due to over-specificity. If such a path exists, flag the rubric.
   - **Adversarial reverse-coverage:** for each rubric, verify a prompt ask exists for it. Flag any rubric that goes beyond the prompt.
   - **Adversarial Process check:** for each Process rubric (if any), re-apply the three-condition test. Flag any that fails.
   - **Adversarial atomicity:** for each rubric, ask "could this fail for two unrelated reasons?" If yes, flag for split.
   - Output to `_aux/Council_Reports/S3_B_adversarial.md`. Block on any Major / Moderate issue.

8. **Loop.** Apply fixes, re-run validator, re-run BOTH councils. Loop until clean.

9. **Coverage matrix.** Write `_aux/Reasoning/Rubric_Coverage_Matrix.md` mapping every prompt sentence → OE step(s) → rubric(s). Confirm no gaps (every prompt ask has a rubric) and no surplus (every rubric ties back to a prompt ask).

## Exit criteria

- `7_Rubrics.json` exists, all rubrics agent-centric, outcome > process, no tool names in titles.
- Validator returns PASS for the rubrics phase.
- Council A returns GO with every concrete value grounded.
- Council B returns GO with every QC rubric sub-dim at 5 and zero adversarial hits.
- Council B-B3 returns projected tool-call count >= 40 — the rubric set must align with a trajectory that produces 40+ tool calls.
- Council B-B4 returns "all Hardness levers from `_aux/Hardness_Plan.md` are covered by at least one Outcome rubric whose value depends on traversing the lever".
- Coverage matrix in place.
