## Procedure

1. **Decompose the prompt.** Read `5_Prompt.txt` sentence by sentence. List every explicit ask and every implicit ask. Map each to:
   - Discovery steps (search emails, query SAP, list exceptions, read documents).
   - Write actions (send, post, create, update, upload, schedule).

2. **Verify each step against the per-task universe.** Every record the OE points to must actually be in `_aux/Universe_Split/`. Every tool name must be in `8_Server_Tools_Details.json`. Every parameter name must be a real parameter for that tool (watch the traps: `content` not `body` for email, `payload` not `text` for Slack, `issueId` for Linear comments, etc.).

3. **Draft `6_Oracle_Events.txt`.** Numbered prose: `OE1: ...`, `OE2: ...`, sequential. Each OE covers what action / discovery, which tool, which parameters with concrete expected values from the per-task universe, and (when relevant) what the agent should conclude.

4. **Run the validator (cheap gate).**
   ```
   python Validators/validate.py --phase oe --task Tasks/<TASK_DIR>
   ```
   Read `_aux/Validator_Reports/oe.md`. Fix every fail. Re-run until clean.

5. **Spawn Council A — Grounding.** The sub-agent walks every OE and verifies the cited tool name, parameter names, expected values, IDs, and amounts against `_aux/Universe_Split/` and `8_Server_Tools_Details.json`. Block on any mismatch. Output to `_aux/Council_Reports/S2_A_grounding.md`.

6. **Spawn Council B — Adversarial QC.** The sub-agent scores OE Completeness and OE Accuracy from the QC spec. Then runs forward + reverse coverage:
   - **Forward:** every sentence in the prompt maps to ≥1 OE step.
   - **Reverse:** every OE step maps to a real prompt ask.
   - Flags any prompt requirement that has no OE step (Completeness fail) or any OE step that goes beyond the prompt (Scope creep).
   Output to `_aux/Council_Reports/S2_B_adversarial.md`. Block on any Major issue.

7. **Loop.** Apply fixes, re-run validator, re-run BOTH councils. Loop until clean.

8. **Strict veteran audit (MANDATORY, auto-fire).** Spawn an audit sub-agent (`oracle`) using the prompt template in `Reference/Sessions/AUDIT.md` with `--phase oe` configured. Save the report to `_aux/Council_Reports/AUDIT_oe.md`. AUDIT applies the STRICTEST QC interpretation — 5/5 is the only acceptable score on every applicable sub-dim (no NON-FAIL band justifications), density bar is 50+ midpoint (not 40 floor), every "should" reads as "must", every OE step must trace end-to-end to a prompt sentence AND to a Fact_Ledger atom. This catches what Council A + B missed.

   | Verdict | Action |
   |---|---|
   | `PASS (STRICT)` | Proceed to step 9 (final report). |
   | `REVISE` | Apply the audit's per-issue fixes IN PLACE in `6_Oracle_Events.txt`. Re-run step 4 (validator) + step 5 (Council A) + step 6 (Council B) + step 8 (AUDIT) against the revised OE list. Iteration cap: **3 REVISE rounds**. After 3, end your response with the full audit issue list and escalate to the user. Any AUDIT finding flagged `PROPAGATE TO S1` (root cause in the prompt) STOPs immediately and the operator must re-run `PIPELINE S1` against the prompt fix first. |
   | `REBUILD` | The OE list has structural issues a fix-in-place cannot repair (e.g., OE list contradicts the prompt's implicit framing across multiple steps; multiple OE steps reference records that do not exist in `_aux/Universe_Split/`). End your response with the audit's REBUILD recommendation and recommend `PIPELINE REDO — Tasks/<TASK_DIR>` in a fresh chat. |

9. **Final report.** Append a short note to `_aux/Reasoning/OE_solvability.md`: the OE-to-prompt coverage map, the OE-to-rubric mapping preview (which OEs become Outcome 1.1, 1.2, 2.1, and which are pure discovery steps that no rubric covers), and the AUDIT verdict.

## Exit criteria

- Validator returns PASS for the OE phase.
- Council A returns GO with every tool and parameter verified.
- Council B returns GO with full forward + reverse coverage.
- Council B-B3 returns projected tool-call count ≥ 50 (design target) OR 40-49 with explicit `THIN_DENSITY` justification carried from HARDNESS. Sparse OEs that cannot project to 50+ must be expanded; < 40 BLOCKS.

- Strict veteran audit returns `PASS (STRICT)` (`_aux/Council_Reports/AUDIT_oe.md` present with that verdict). `REVISE` iterations must be resolved within the 3-round cap; `REBUILD` STOPs to REDO; any `PROPAGATE TO S1` finding STOPs to S1 re-run.
