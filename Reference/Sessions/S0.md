# PIPELINE S0 — Setup

Triggered by: `PIPELINE S0 — Tasks/<TASK_DIR>`

## What this phase does

Prepares the per-task working set so every downstream phase runs against a clean per-task source of truth. No deliverables yet, just setup.

## Required inputs (must already exist in `Tasks/<TASK_DIR>/`)

| File | Source |
|---|---|
| `1_Business_Function.txt` | user-pasted |
| `2_Persona.txt` | user-pasted |
| `3_UniverseDataForThisTask.json` | user-pasted (the per-task pgweb export) |

## Procedure

1. **Extract the Persona Brief.** Read `Brookfield_Base_Universe/2_Persona_Briefs.md`, find the section matching the persona named in `2_Persona.txt`, and copy that section verbatim to `Tasks/<TASK_DIR>/PersonaBrief.txt`. Do not paraphrase, do not add anything that is not in the source.

2. **Split the per-task universe.** Run:
   ```
   python Validators/split_universe.py Tasks/<TASK_DIR>
   ```
   This writes `Tasks/<TASK_DIR>/_aux/Universe_Split/<service>.<table>.json` files and `_aux/data_hash.txt` (sha256 of the input). Confirm the script prints the per-source record counts and exits 0.

3. **Build the Universe Index.** Run:
   ```
   python Validators/build_universe_index.py Tasks/<TASK_DIR>
   ```
   This produces `Tasks/<TASK_DIR>/_aux/Universe_Index/`:
   - `service_inventory.md`
   - `entities_personas.md`
   - `key_facts.md`
   - `today_horizon.json`
   - `accounts_per_entity.md`

4. **Build the Fact Ledger.** Run:
   ```
   python Validators/build_fact_ledger.py Tasks/<TASK_DIR>
   ```
   Produces `Tasks/<TASK_DIR>/_aux/Fact_Ledger.json`. This is the authoritative atom surface Council A and the validator use for groundedness. Every email, money amount, ISO date, JE id, exception id, recon id, vendor id, AP invoice id, slack channel id, contact id, persona id, account-per-entity mapping, and fiscal-period status lands here.

5. **Build the Graph Report.** Run:
   ```
   python Validators/build_graph_report.py Tasks/<TASK_DIR>
   ```
   Produces `Tasks/<TASK_DIR>/_aux/Universe_Index/graph_report.md`. Compact people-by-density, periods-by-JE-count, exceptions-by-state, AP-by-vendor map. Used by HARDNESS to pick the densest persona × period intersection for stump-able levers.

6. **Write the setup report.** Create `Tasks/<TASK_DIR>/_aux/S0_Setup_Report.md` summarizing:
   - persona name + role (from PersonaBrief)
   - business function (from `1_Business_Function.txt`)
   - per-task data hash (from `_aux/data_hash.txt`)
   - record count totals from `service_inventory.md`
   - the universe today and last_event_timestamp_seen from `today_horizon.json`
   - a one-line note flagging any records_dated_after_today > 0 with the universe today as context

## Exit criteria

- `PersonaBrief.txt` exists and is non-empty.
- `_aux/Universe_Split/` contains the per-service JSON files.
- `_aux/Universe_Index/` contains all 5 summary files.
- `_aux/S0_Setup_Report.md` is written.
